Intelligente 404-Seite für Wordpress

Mit wachsender Größe eines Blogs lassen sich 404-Fehler nicht vermeiden. Es wird vorkommen, dass jemand falsche Links setzt, dass User auf kaputte Links z.B. in Emails klicken oder das schlichtweg mal ein Posting gelöscht oder wegen Tippfehlern die URL geändert wird.
In jedem dieser Fälle bekommt der geneigte Besucher normalerweise eine recht simple 404-Seite vorgesetzt die ihm mitteilt, dass er nichts gefunden hat. Ideal wäre es natürlich, wenn stattdessen eine hilfreiche Seite angezeigt würde.

Eine Lösung dafür gibt es über das hier kürzlich schon vorgestellte Plugin "Related Posts": Es gibt eine Variante des Plugins die einen speziellen Befehl <?=related_posts_404?> anbietet mit dem aus einem 404-Fehler eine brauchbare Navigation werden kann. Voraussetzung dafür ist allerdings, dass man URL-Rewriting nutzt, denn nur dann kann das Plugin aus der URL ablesen welcher Beitrag gewünscht wurde.
